**本次只实体化 `libs/common`、`pipeline`、`eval` + 仓库级杂项**(硬约束#4「最小脊柱,不铺空包」)。
上面标 ❌ 的 `libs/rag|orchestration|doc_compare|report_gen` 与 `services/*` **是约定,不是已建**——
等各自第一份真实代码或第二个消费方出现再创建(见 §3 抽包触发)。

### 1.1 决策⑤修订:config/ + seeds/ 置于 repo 根(非 pipeline 成员内)

决策⑤原定 config/seeds 迁入 pipeline。**升格 Step 5 实测推翻**:flat 布局下成员目录 `pipeline/` 与
import 包 `pipeline` 同名,若 `config/` 置于 `pipeline/` 内,则当 `cwd=repo 根`在 `sys.path` 时
`pipeline/config` 被当**命名空间包遮蔽 `pipeline.config` 模块** → `ImportError`,实测 break
`python -m alembic`(=`demo up` 建库步)。重装 editable 无效(其 finder 追加在 `sys.meta_path` 尾,
cwd 路径优先)。**故 config/ + seeds/ 作 workspace 级置 repo 根**(与 `alembic/`、`compose.yaml` 一致);
`config.py` 模块仍属 pipeline(决策⑤「config 域归 pipeline」的意图由模块归属体现)。**待 zy 确认。**

## 2. 包间依赖方向(DAG,无环)

```
eval ──→ pipeline ──→ common
                       ▲
        (verify/* 用 StageContext/MilvusIO/corpus_rows)
```

- `common`:契约承重层,**不依赖任何上层**。
- `pipeline → common`。
- `eval → pipeline → common`(决策②:verify 组件实 import pipeline 内部)。
- **`pipeline` 不得在 import 期依赖 `eval`**:cli/web/finalize 调用 eval 验证组件一律**函数级懒导入**
  (决策:断 `pipeline⇄eval` 环;实测 `import pipeline.*` 零拉入 eval,声明依赖无环;eval 仅 verify/report
  运行时被懒载,monorepo 同装可接受)。新增对 eval 的调用必须沿用懒导入。
- 工具链:isort `known-first-party = [common, pipeline, eval]`;pytest `pythonpath/testpaths` 列三成员。

## 3. 抽包触发条件(何时把模块抽成 `libs/` 共享包)

**铁律:出现「第二个消费方」才抽共享包**(单消费方留在其包内,避免过早抽象与空包)。

| 候选共享包 | 抽取触发 |
|---|---|
| `libs/rag` | 查询智能体 + 比对智能体都要复用同一检索/混合查逻辑(第二消费方) |
| `libs/orchestration` | Temporal 落地且 ≥2 个服务复用同一编排骨架 |
| `libs/doc_compare` | 比对领域逻辑被报告生成等第二方复用 |
| `libs/report_gen` | 报告生成被 ≥2 场景(审计规范 / 审计报告)复用 |
| `services/<name>` | 出现需独立部署的服务,或 web 工作台需脱离 pipeline 独立运行 |

契约层 `libs/common` 是例外:它天然是全员单一来源,已先建。

## 5. codegraph 落地(本机已建索引;信创生产走离线步骤)

**本机(开发)已联网落地**(zy 免信创约束):`codegraph` 1.0.1 全局装好;`codegraph init` 建基线索引
= **108 files / 1,663 nodes / 5,247 edges**(node:sqlite full WAL,`.codegraph/` 已 gitignore、不入库);
匿名遥测已 `codegraph telemetry off`。git 钩子见 `.githooks/`(`core.hooksPath=.githooks`,各钩子
`command -v codegraph || exit 0` 守卫):`post-merge`/`post-checkout`/`pre-push` → `codegraph sync`(增量刷新)。

**信创生产(无外网)离线步骤**:
1. **外网机**:`npm pack @colbymchenry/codegraph` 产出 tarball,带入内网(vendor 或本地 npm 源)。
2. **内网**:用 tarball 本地安装 → 可选 `codegraph install`(把 codegraph MCP 装进 agent,改本地 agent 配置)。
3. `codegraph init`(默认即建索引;整树搬迁后建干净基线)→ 需重建时 `codegraph index --force`。
4. 钩子已在 `.githooks/`(克隆后 `git config core.hooksPath .githooks` 启用)。
5. **智能选测**:CI 或 pre-push 可用 `codegraph affected <changed-files>` 圈出受影响测试只重跑(替换 CI 现占位的全量 `pytest`);本次未强制挂到 pre-push 以免阻断 push,留作 CI 优化项。

> 注:`codegraph install`(MCP 装进 Claude Code 等 agent)会改本机 agent 配置 + 写 auto-allow 权限,
> 属可选、独立于"建索引";本次**未执行**(仅建了索引 + 钩子),按需再装。

### 5.1 三项信创确认(待 zy 拍板,影响 codegraph 可用性)
